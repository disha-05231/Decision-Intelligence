## 📌 Project Overview

The Human Decision Intelligence Engine (HDIE) is designed to help organizations understand customer decision-making behavior using Machine Learning and Behavioral Analytics.

The system:

* Analyzes customer clickstream interactions
* Generates behavioral features
* Segments customers into meaningful groups
* Predicts purchase likelihood
* Explains model decisions using Explainable AI (SHAP)
* Provides business recommendations
* Visualizes insights through an interactive Streamlit dashboard

# 🔧 Major Challenges & Fixes

## Challenge 1

Problem:

Performance page contained hardcoded metrics.

Fix:

Created dynamic metric loading using:

[reports/model_metrics.json](reports/model_metrics.json)

Result:

Dashboard always reflects actual model performance.

---

## Challenge 2

Problem:

SHAP analysis failed due to feature mismatch.

Error:

'abandonment_risk' not in index

Fix:

Aligned explainability features with training features.

Result:

Successful explainable AI pipeline.

---

## Challenge 3

Problem:

Top feature rankings were hardcoded.

Fix:

Calculated feature importance dynamically using SHAP values.

Result:

Explainability page became fully model-driven.

---

## Challenge 4

Problem:

requirements.txt corruption (NUL characters).

Fix:

Manually rebuilt requirements file.

Result:

Deployment-ready environment.
